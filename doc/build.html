# 打包

首先要求所有的环境都安装无误
可以先执行一下命令做检查

weexplus env

# 何时需要打包

  • 想给用户最终的产物
  • 添加或删除了插件
  • 修改了weexplus.json
  • 自己修改了原生代码

# 正式包与开发包

  • 首页有设置刷新的按钮即为开发包否则为正式包

  • 由weexplus.json中debug字段控制，true为开发包，false为正式包 打最终的正式包一定记得先把debug改false

  • 开发包会默认加载上次扫码之后保存的地址

  • 生产包会默认加载entry.vue页面，但是加载entry页面的时候只显示启动页，所以请在entry.vue中写跳转到真正首页的逻辑代码

# 设置证书

默认在android目录内已经制作了一份证书，不过大家创建项目后都用的这一个，所以打正式包的时候务必创建自己的证书覆盖掉这个

操作视频如下

制作一份自己的证书名字为keystore.jks ，替换掉原位置的证书,然后去gradle.properties里面修改别名和密码 此步骤只用操作一次

# 修改配置

  • 修改weexplus.json中debug的值，如果是正式包请改为false
  • 修改weexplus.json中showError的值，如果是正式包请改为false

# 自动打包

项目根目录下执行

weexplus publish android

执行完之后会生成apk文件，即为最终的输出包

# 手动打包

如果上面的命令打包失败，可以手动打包

1. weexplus install

2. 点击android studio菜单上的build->build signed apk
